# 视频转顺序帧工具

一个纯本地浏览器工具。把视频拖进页面后，会按固定 `24 FPS` 抽帧，输出自动命名的 `JPG` 顺序帧，并打包为 `ZIP` 下载。

## 运行

### 方式 1：直接双击启动

双击项目根目录里的 `start.bat`。

它会自动：

1. 安装依赖（首次运行时）
2. 启动本地网页
3. 自动在浏览器打开工具页面

### 方式 2：命令行启动

```powershell
npm install
npm run dev -- --open
```

## 使用

1. 打开网页
2. 选择压缩档位：`150K`、`500K` 或 `不压缩`
3. 把视频拖进页面，或点击选择视频
4. 等待浏览器本地抽帧、压缩和打包
5. 自动下载 `ZIP` 文件

## 输出规则

- 固定按 `24 FPS` 导出
- 帧命名格式：`原视频名_000001.jpg`
- 压缩包命名格式：`原视频名_24fps_frames.zip`
- 保持原分辨率不变
- `150K` 档：目标 `150 KB`，必要时允许到 `200 KB`
- `500K` 档：目标 `500 KB`
- `不压缩` 档：高质量 `JPG` 直出，不做体积限制

## 性能优化

- 主路径改成“高速播放采样”而不是逐帧 seek
- ZIP 改成 `STORE`，避免对已经压缩过的 JPG 再做二次压缩
- JPG 压缩改成带记忆的快速自适应策略，减少每帧重复试算次数
- 如果高速路径失败，会自动回退到稳妥的逐帧 seek 路径

## 说明

- 整个过程只在本地浏览器完成，不会上传视频
- 推荐浏览器：`Chrome / Edge`
- 推荐格式：`MP4(H.264)`、`WebM`
- 如果原视频低于 `24 FPS`，导出的 `24 FPS` 序列里会出现重复帧，这是正常现象
- “不压缩”仍然是 `JPG` 输出，不等于真正无损；如果后面需要无损导出，需要单独增加 `PNG`
