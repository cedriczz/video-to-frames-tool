# 视频转顺序帧工具设计

## 目标

做一个纯本地浏览器网页工具。用户把视频拖进页面后，工具在本地浏览器里按固定 `24 FPS` 抽帧，自动命名为顺序 `JPG`，再打包成 `ZIP` 下载。

## 方案

- 前端单页应用，无后端服务
- 使用浏览器原生 `video` 解码视频
- 使用 `canvas` 捕获帧并导出为 `JPG`
- 使用 `JSZip` 在浏览器本地打包
- 默认固定输出 `24 FPS`

## 交互

- 页面支持拖拽视频和点击选择文件
- 选中文件后立即开始处理，不额外增加确认按钮
- 页面显示文件名、分辨率、时长、预计帧数、处理进度
- 处理完成后自动触发 ZIP 下载

## 命名与导出

- 帧文件名格式：`原视频名_000001.jpg`
- ZIP 名称格式：`原视频名_24fps_frames.zip`
- ZIP 内部包含一个同名目录，目录内是所有顺序帧

## 24 FPS 逻辑

- 按 `1/24` 秒的时间间隔采样
- 总帧数使用 `ceil(duration * 24)` 计算，保证整个视频时间轴被覆盖
- 原视频高于 `24 FPS` 时，相当于降采样到 `24 FPS`
- 原视频低于 `24 FPS` 时，仍然导出 `24 FPS` 编号序列，但会出现重复画面

## 风险

- 依赖浏览器原生解码，最佳兼容格式是 `MP4(H.264)` 和 `WebM`
- 大视频会带来更高的内存和处理时间成本
- 这不是专业 NLE 级逐帧解码方案，而是适合“拖入即导出”的轻量本地工具
